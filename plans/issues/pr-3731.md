## Why this matters

A child of a draggable element calling `event.stopPropagation()` in its
`onPointerUp` prevents `PanSession`'s window-level bubble-phase `pointerup`
from firing, so the element keeps following the cursor after release (#2794,
open since 2024). The PR moves the session's end listeners to the capture
phase, which user code cannot intercept, and fixes a latent listener leak.

## Current state

The PR changes 4 files (+61/−9), all untouched on main since the merge-base
(rebase expected conflict-free; the branch is ~35+ commits behind):

- `packages/framer-motion/src/gestures/pan/PanSession.ts` — all three session
  listeners get `{ passive: true, capture: true }`:
  ```ts
  const eventOptions = { passive: true, capture: true }
  this.removeListeners = pipe(
      addPointerEvent(this.contextWindow, "pointermove", this.handlePointerMove, eventOptions),
      addPointerEvent(this.contextWindow, "pointerup", this.handlePointerUp, eventOptions),
      addPointerEvent(this.contextWindow, "pointercancel", this.handlePointerUp, eventOptions))
  ```
- `packages/motion-dom/src/events/add-dom-event.ts` — leak fix, load-bearing:
  the returned remover now passes `options` to `removeEventListener` (capture

  (`pointerup`/`pointercancel`) moved to capture via
