## Why this matters

The motion-dom stats system ships ~580 B gz of per-frame counters and summary
reporting that nothing in the published packages consumes. This PR deletes the
unused internals while preserving the one real consumer surface
(`statsBuffer.value.layoutProjection`, used by the dev HTML projection asserts
and `framer-motion/src/projection.ts` re-exports). It also makes PR #3742 (a
micro-refactor of code this PR deletes) obsolete.
