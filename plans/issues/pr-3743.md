## Why this matters

`motion/react` re-exports `motion`/`m` from `framer-motion` twice (named
re-export + `export *`), the only structural difference from importing
`framer-motion` directly — and #3741 reports that difference OOMs Turbopack's
dev server on Windows. The PR replaces the named re-export with a namespace
import + local consts (local declarations shadow `export *` per spec),
preserving IDE auto-import (#3432). The catch: the OOM is environment-specific
and has never been reproduced by the author, and the reporter hasn't
confirmed. Per this repo's norm ("no repro → no fix"), this PR must not merge
on speculation alone — it needs either reporter confirmation or a locked-in
structural invariant plus explicit maintainer acceptance of the
environment-specific-fix exception in CLAUDE.md ("a test that can't fail
without the fix is acceptable when the bug is environment-specific").
