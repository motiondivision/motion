## Why this matters

`snapToCursor` is currently only reachable via
`dragControls.start(e, { snapToCursor: true })`, which conflicts with
declarative props like `dragSnapToOrigin`. Issue #2677 asks for a
`dragSnapToCursor` prop; this PR adds it. The implementation is tiny and
additive, and CI is fully green — but the PR is the least merge-ready of the
small ones: 128 commits behind (merge-base 2026-04-08, predating three May
changes to `VisualElementDragControls.ts`), and its only tests are spy-based
unit tests that verify option *plumbing*, not the actual snap-to-cursor
behavior — which is exactly the pointer-position-dependent visual behavior
CLAUDE.md says needs Cypress.
