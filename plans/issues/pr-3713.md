## Why this matters

`scroll()` only supports `offset`, which clamps progress to 0/1 outside the
range and keeps styles applied â€” blocking e.g. `:hover` styles outside the
animated range. Native WAAPI `rangeStart`/`rangeEnd` with `fill: "auto"` makes
the animation inactive (`progress === null`) outside the range. Community user
bakura10 explicitly asked for exactly this combination ("provide rangeStart,
rangeEnd and offset (for the JS fallback) â€¦ best of both worlds", comment on
closed #3646, 2026-03-25, citing Safari perf issues). The PR implements it
cleanly; it is draft solely because the author wouldn't re-submit a
silently-closed change without an explicit decision.
