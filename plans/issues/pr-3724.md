## Why this matters

This PR's verdict hinges on one unanswered question: **does the fix actually
fix the bug in real Chrome?** Its own test plan's final checkbox — "Verify in
Chrome with the original CodeSandbox repro" — is unchecked, and the Cypress
spec admits it "documents the expected behaviour rather than reliably failing
on the unfixed codebase" (the quirk doesn't reproduce in Electron). Meanwhile
the fix is an unconditional `offsetWidth` read (forced reflow) in the
post-layout-animation render path of **every** element in **every** browser:

```ts
// packages/motion-dom/src/projection/node/create-projection-node.ts,
// right after `this.hasProjected = false`:
if (this.instance) {
    void (this.instance as unknown as HTMLElement).offsetWidth
}
```

Unverified workaround + hot-path perf cost is a bad trade by default. The
repo's own rule applies: no repro → no fix. But a real-Chrome verification
path exists (Cypress `--browser chrome` / Playwright Chromium) and was never
used — so verify first, then merge-or-close on evidence.

### Step 2 (only if Step 1 verified the fix): scope the reflow, then merge

The unconditional `offsetWidth` read penalizes every layout animation to fix a
Reorder-specific Chrome quirk. Scope it before merging — gate the read so it
only runs when warranted. Preferred: only when the node's instance is inside a
scrollable ancestor being observed is NOT knowable cheaply, so the pragmatic
scope is to gate on Chrome (`typeof navigator !== "undefined" && /Chrome/.test(navigator.userAgent)`
is NOT this repo's style — instead check for an existing capability/UA
pattern in `packages/motion-dom/src/utils/` and follow it; if none exists,
keep the read unconditional but move it behind the existing
`this.options.alwaysMeasureLayout`-style option check ONLY if semantics
allow). If no acceptable scoping exists, present both variants (unconditional
vs UA-gated) as a PR comment for maintainer choice and mark the README row
BLOCKED. Also fix the Greptile nits (single cast via `this.instance as HTMLElement`
if types allow; `time.now()` in the test page per CLAUDE.md Timing note).
Then rebase, fail-first-check the spec in Chrome (Step 1.1 loop), push, green
CI, `gh pr merge 3724 --squash`.
