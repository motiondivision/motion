## Why this matters

\#3747 and #3748 are mutually exclusive total rewrites of the same single
source file (`packages/motion-dom/src/layout/LayoutAnimationBuilder.ts`) plus
the same fixture JSON — guaranteed merge conflict if both stay open. #3748 is
the strictly stronger, newer iteration (no shared commits; verified disjoint:
3747 = `8e1391e4c`,`97132c393`; 3748 = `498aa804a`,`87d61a8e7`):

- **Batching model**: #3747 batches implicitly via the shared root's
  microtask-deduped `didUpdate()`, so builder A's synchronous `updateDom` runs
  *before* builder B snapshots — same-tick calls on intersecting trees can
  corrupt each other's origin measurements. #3748 snapshots all builders
  before any DOM mutation and reconciles cross-builder; its
  `parallel-calls-intersecting.html` fixture targets exactly this and fails on
  #3747's model.
- **Fixture coverage**: #3748's fixtures cover #3747's
  (`parallel-calls-disjoint` ≈ `parallel-siblings`,
  `parallel-calls-intersecting` ⊃ `parallel-intersecting`) plus two #3747
  lacks. The ONLY #3747-exclusive artifact is `fire-and-forget.html` — ported
  to #3748 by pr-3748 Step 2 before this close executes.
- **CI**: #3747 is BLOCKED on failing required checks (flaky drag/layout specs
  across 4 documented runs); #3748 is fully green.
