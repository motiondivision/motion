## Why this matters

When a child of `AnimatePresence` triggers its exit and then its only motion
component unmounts on the next render, `PresenceChild` never fires
`onExitComplete`, so the exiting wrapper is stuck in the DOM forever (issue
#3243). The PR fixes this and carries a regression test that reproduces the
reporter's exact scenario and fails without the fix. This is the most
merge-ready open PR: real repro, failing-first test, zero drift. It only needs
a rebase and a green CI run.

## Current state

- The PR changes 2 files (+72/−2):
  - `packages/framer-motion/src/components/AnimatePresence/PresenceChild.tsx` —
    `isPresentRef`/`onExitCompleteRef` are now updated in
    `useIsomorphicLayoutEffect` (concurrent-render safety), and the `register`
    cleanup fires exit completion when the last child deregisters during exit:
    ```ts
    return () => {
        presenceChildren.delete(childId)
        !isPresentRef.current &&
            !presenceChildren.size &&
            onExitCompleteRef.current?.()
    }
    ```

    regression test: child swaps its `motion.div` (10s exit) for a plain
