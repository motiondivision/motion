## Why this matters

A variant returning a literal `transform` string (e.g. a Windows-style
draggable window using `dragControls` + `dragListener={false}`) disables drag
movement entirely: `buildHTMLStyles` sees `latestValues.transform` and skips
building the transform from the drag-driven `x`/`y` values (#2807, open since
2024). This is the most test-complete fix PR open: unit test + Cypress E2E
mirroring the issue, verified failing on main, green on both React 18 and 19
in CI.
